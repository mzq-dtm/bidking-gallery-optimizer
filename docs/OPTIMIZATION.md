# 优化算法

本文说明银币产出优化目标、展示价值、最大费用流、同产出效率少移动重排和移动步骤生成。

## 优化目标

《竞拍之王》中，展示柜会依据柜内藏品的价值每小时产出银币。每小时银币产出效率由全部展示柜中藏品的折后总价值决定。

因此，本工具的首要目标是最大化折后总价值，也就是最大化每小时银币产出效率。在达到最高产出效率后，第二目标是减少调整展示柜所需的移动步骤。

## 折后价值计算

折后总价值只计算展示柜中的藏品，背包物品不直接产生银币，但会作为优化摆放时的候选库存。

同名藏品使用递减价值：

| 件数 | 价值比例 |
|---:|---:|
| 第 1 件 | 100% |
| 第 2 件 | 50% |
| 第 3 件 | 25% |
| 第 4 件 | 13% |
| 第 5 件 | 7% |
| 第 6 件及以后 | 4% |

折后价值只取决于同名藏品在全部展示柜中的总数量，与具体柜子和位置无关。程序以折后总价值作为银币产出效率的优化指标。

## 第一阶段：最大化银币产出效率

`optimize_cabinet.build_optimized_cabinets()` 使用最大费用流选择折后总价值最高的展示方案，从而最大化每小时银币产出效率。

网络结构：

```text
源点 -> 藏品边际价值节点 -> 可放展示柜节点 -> 汇点
```

同名藏品会按出现次数拆成不同的边际价值节点。例如基础价值为 `1000`：

```text
第 1 件：1000
第 2 件：500
第 3 件：250
第 4 件：130
第 5 件：70
第 6 件：40
第 7 件及以后：40
```

约束：

- 物品只能连接到类别匹配的展示柜。
- 每个物品副本最多使用一次。
- 每个展示柜到汇点的容量为 15。

算法持续寻找正收益最高的增广路，直到不存在正收益路径。

## 第二阶段：同产出效率下减少移动

第一阶段确定达到最高银币产出效率时“最终展示哪些藏品及数量”，但不保证具体柜位产生最少移动。

`optimize_plan.minimize_steps_for_same_value()` 会把第一阶段结果压缩成目标数量：

```text
flow_cabinets -> target_counts
```

第二次网络结构：

```text
源点 -> 目标藏品节点 -> 当前来源节点 -> 展示柜节点 -> 汇点
```

| 边 | 容量 | 单位费用 | 含义 |
|---|---:|---:|---|
| 源点到藏品 | 目标数量 | 0 | 必须安排的数量 |
| 藏品到当前来源 | 当前库存数量 | 0 | 选择具体物品来源 |
| 来源到原展示柜 | 来源数量 | 0 | 保留在原柜 |
| 来源到其他展示柜 | 来源数量 | 1 | 产生一次位置变化 |
| 展示柜到汇点 | 15 | 0 | 展示柜容量 |

第二阶段求最小费用满流：

- 满流确保所有目标藏品都被放入展示柜。
- 最小费用尽量保留原柜中的同名藏品。

在当前展示数量和目标展示数量固定时：

```text
移动步骤数
= 当前展示数量 + 目标展示数量 - 2 * 原柜保留数量
```

因此，最大化原柜保留数量等价于减少最终移动步骤。

## 回退策略

旧的 `minimize_moves_for_same_value()` 保留为回退路径。

出现以下情况时会回退：

- 最小费用满流无法安排全部目标藏品。
- 重排后的展示价值与第一阶段最大价值不一致。

这可以避免部署不完整或价值下降的方案。

## 移动步骤

`build_rearrangement_moves()` 将方案拆成两个阶段：

```text
先腾出展示柜位置：
...
再放入推荐展品：
...
```

先移出、后放入可以避免向已满展示柜继续放置物品。

移动步骤按展示柜顺序排列，同一展示柜内按物品名称排序。

## 最终背包

`build_optimized_backpack()` 使用库存守恒计算最终背包：

```text
最终背包 = 当前展示柜 + 当前背包 - 优化后展示柜
```

优化过程不会凭空创建或丢失物品。
