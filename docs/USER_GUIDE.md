# 用户指南

本文介绍竞拍之王展示柜优化工具的界面布局、编辑方式和分析功能。展示柜内藏品的折后总价值决定每小时银币产出效率，该工具通过优化战利品的摆放方案来最大化银币产出效率。

## 启动程序

推荐使用 Python 3.11 或更高版本：

```bash
python main.py
```

也可以使用兼容入口：

```bash
python cabinet_gui.py
```

如果当前 Python 环境缺少 Tkinter，程序会提示安装对应的 Tkinter 组件。

## 主界面

主窗口固定为 `2000x900`。

内容区按 4 行 3 列排列：

- 11 个展示柜占据前 11 个位置。
- 背包位于右下角的第 12 个位置。
- 顶部显示当前展示柜折后总价值，该数值决定每小时银币产出效率。
- 顶部提供 `需保留的物品`、`分析展示柜` 和 `分析背包替换` 按钮。

每个展示柜会显示：

- 展示柜名称。
- 当前数量和 15 件容量上限。
- 按名称排序的藏品列表。
- `编辑展品` 按钮。

背包区域会显示：

- 当前物品总数。
- 四列表格，每行展示两组“物品名”和“数量”。
- `清空背包` 和 `编辑展品` 按钮。

## 编辑展示柜

点击展示柜的 `编辑展品`：

- 编辑窗口提供 15 个下拉框。
- 下拉选项只包含符合该展示柜类别要求的藏品。
- 保存后，藏品会按名称排序。
- 排序结果会同时应用到 GUI 和 `display_cabinets.toml`。

## 编辑背包

点击背包的 `编辑展品` 会打开独立编辑窗口：

- 当前内容使用四列表格展示。
- 每行包含两组“物品名”和“数量”。
- 点击任意一组，会将对应物品和数量回填到下方编辑区。
- 使用下拉菜单选择物品。
- 在数量输入框中输入大于等于 `1` 的整数。
- `添加/更新` 会新增物品或覆盖该物品的数量。
- `删除当前` 会删除当前回填的物品。
- `保存` 后才会写入 `backpack_items.json`。

背包物品总数不设上限。

## 清空背包

点击 `清空背包`：

1. 程序弹出确认窗口。
2. 确认后清空 `backpack_items.json`。
3. GUI 自动刷新。

如果背包已经为空，程序会直接提示无需清空。

## 展示柜分析

点击 `分析展示柜` 后，程序会汇总所有展示柜中的藏品。

表格包含：

- `藏品`
- `数量`
- `最低生效价值`

结果按最低生效价值升序排列，便于找到当前贡献最低的展品。

## 需保留的物品

点击 `需保留的物品` 后，程序会：

1. 找到当前展品中的最低生效价值。
2. 计算每种藏品再加入一件时的实际贡献价值。
3. 列出贡献价值高于当前最低值的藏品。

表格包含：

- `藏品`
- `当前数量`
- `加入后实际价值`

结果按加入后的实际价值降序排列。

该功能用于快速筛选候选物品，不考虑展示柜类别限制。完整替换仍应使用背包替换分析。

## 背包替换分析

点击 `分析背包替换` 后，程序会从当前展示柜和背包的总库存中寻找更高价值的展示方案。

推荐方案会优先最大化折后总价值，也就是最大化每小时银币产出效率；在总价值相同时，再尽量减少物品移动步骤。

分析结果包含：

- 当前总价值
- 推荐后总价值
- 增加价值
- 移除的物品
- 加入展示柜的物品
- 背包中未使用的物品
- 可执行移动步骤

分析阶段不会修改文件。只有点击 `确定部署` 后，程序才会更新：

- `display_cabinets.toml`
- `backpack_items.json`

随后 GUI 会重新加载并显示最新状态。
