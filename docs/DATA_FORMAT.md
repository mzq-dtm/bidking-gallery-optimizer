# 数据格式

程序从项目根目录读取三个数据文件。

## display_cabinets.toml

保存展示柜定义与当前藏品：

```toml
[["展示柜"]]
"名称" = "医疗用品柜"
"可放类别" = ["医疗用品"]
"藏品" = ["百年人参", "超声波诊断仪"]
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `名称` | 字符串 | 展示柜名称 |
| `可放类别` | 字符串数组 | 允许放入的藏品类别 |
| `藏品` | 字符串数组 | 当前展示的藏品名称 |

规则：

- 每个展示柜最多放 15 件藏品。
- 藏品至少有一个类别与展示柜允许类别相同，才可以放入该柜。
- 读取、编辑或写回文件时，每个柜子的藏品都会按名称排序。

## item_category.json

保存藏品目录、基础价值和类别：

```json
[
  {
    "藏品名": "百年人参",
    "价值": 1000,
    "藏品类别": ["医疗用品", "饮食烹饪"]
  }
]
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `藏品名` | 字符串 | 藏品的唯一名称 |
| `价值` | 整数 | 第一件藏品的基础价值 |
| `藏品类别` | 字符串数组 | 藏品所属类别 |

`item_category.csv` 是数据参考文件，当前程序运行时读取的是 `item_category.json`。

## backpack_items.json

保存背包中的物品。文件使用字符串数组，同名物品通过重复记录表示数量：

```json
[
  "百年人参",
  "百年人参",
  "AI服务器"
]
```

上例表示：

- 百年人参：2 件
- AI服务器：1 件

规则：

- 背包物品数量不设上限。
- 背包物品不直接计入展示总价值，也不直接产生银币。
- 背包物品会作为展示柜优化的候选库存。

## 文件写入

GUI 保存时：

- 展示柜通过 `utils.write_cabinets()` 写入 TOML。
- 背包通过 `utils.write_backpack_items()` 写入 JSON。
- 展示柜 TOML 写入后会再次解析，以确认文件格式有效。
- 部署优化方案时，展示柜和背包文件会一起更新。
